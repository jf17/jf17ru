Mediator (Посредник) :

 Паттерн заставляет объекты общаться не напрямую друг с другом, а через отдельный объект-посредник, который знает, кому нужно перенаправить тот или иной запрос.



Для чего используется:

 Уменьшение связанности между взаимодействующими объектами. Централизация сложной логики взаимодействия.



Пример использования:

 Диалоговые окна с множественными элементами управления, авиадиспетчерская служба, чат-комнаты.



Простой пример из жизни:

 Как авиадиспетчер в аэропорту: самолёты не общаются напрямую друг с другом, а все команды проходят через диспетчерскую вышку.
