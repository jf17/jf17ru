Interface Bloat (Распухший Интерфейс) :

 Антипаттерн создания интерфейсов с избыточным количеством методов. Нарушает принцип разделения интерфейсов (Interface Segregation Principle).



Признаки проблемы:

 Интерфейсы с множеством методов, классы реализуют только часть методов интерфейса, пустые реализации методов, сложность тестирования.



Как избежать:

 Разделение больших интерфейсов на более мелкие, применение принципа разделения интерфейсов, создание специализированных интерфейсов.



Простой пример из жизни:

 Как универсальный пульт с 200 кнопками: большинство функций вам не нужны, но вы вынуждены "поддерживать" весь интерфейс, даже если используете только 5 кнопок.
