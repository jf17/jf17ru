Accidental Complexity (Случайная Сложность) :

 Антипаттерн введения ненужной сложности в систему из-за неправильных технических решений, а не из-за сложности самой задачи.



Признаки проблемы:

 Сложность не связана с бизнес-логикой, избыточные технологии, неподходящие инструменты, переусложнённая архитектура.



Как избежать:

 Выбор подходящих технологий, фокус на существенной сложности, рефакторинг, упрощение архитектуры, использование проверенных решений.



Простой пример из жизни:

 Как использование экскаватора для посадки цветка: задача простая (выкопать ямку), но выбранный инструмент создаёт ненужную сложность и проблемы.
