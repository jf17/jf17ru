Observer (Наблюдатель) :

 Поведенческий паттерн, который определяет зависимость типа "один ко многим" между объектами таким образом, что при изменении состояния одного объекта все зависящие от него объекты уведомляются и обновляются автоматически.



Для чего используется:

 Уведомление множества объектов об изменениях в другом объекте. Реализация слабой связанности между объектами. Создание системы событий и подписок. Обновление представлений при изменении модели данных.



Пример использования:

 EventListener в GUI приложениях, MVC архитектура, системы уведомлений, подписки на изменения в базе данных, реактивное программирование.



Простой пример из жизни:

 Как подписка на газету: когда выходит новый номер, издательство автоматически отправляет его всем подписчикам. Подписчики не проверяют постоянно, есть ли новый номер - они просто получают уведомление.
