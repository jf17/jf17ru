Chain of Responsibility (Цепочка Обязанностей) :

 Паттерн позволяет передавать запросы последовательно по цепочке обработчиков. Каждый последующий обработчик решает, может ли он обработать запрос сам и стоит ли передавать запрос дальше по цепи.



Для чего используется:

 Обработка запросов несколькими обработчиками без жёсткой привязки отправителя к получателю.



Пример использования:

 Обработка HTTP-запросов в веб-фреймворках, системы логирования с разными уровнями.



Простой пример из жизни:

 Как обращение в службу поддержки: сначала отвечает бот, если не может помочь - передаёт оператору первой линии, затем специалисту, потом руководителю.
