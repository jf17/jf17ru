Magic Button (Волшебная Кнопка) :

 Антипаттерн, при котором одна кнопка или метод выполняет слишком много разнородных функций. Нарушает принцип единственной ответственности.



Признаки проблемы:

 Один элемент интерфейса выполняет множество функций, сложная логика в одном методе, непредсказуемое поведение для пользователя.



Как избежать:

 Разделение функций на отдельные кнопки/методы, применение принципа единственной ответственности, чёткое именование элементов.



Простой пример из жизни:

 Как швейцарский нож: одна кнопка открывает и нож, и ножницы, и отвёртку. Непонятно, что получишь, пока не нажмёшь. Лучше иметь отдельные инструменты.
