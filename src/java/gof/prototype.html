Prototype (Прототип) :

 Паттерн Prototype позволяет копировать объекты, не вдаваясь в подробности их реализации. Все классы-прототипы должны иметь общий интерфейс, позволяющий копировать объекты.



Для чего используется:

 Клонирование объектов без привязки к их конкретным классам. Уменьшение количества подклассов.



Пример использования:

 Интерфейс Cloneable в Java, копирование сложных объектов конфигурации.



Простой пример из жизни:

 Как ксерокопирование документа: у вас есть оригинал, и вы делаете его точные копии. Каждая копия независима, но содержит ту же информацию.
