Proxy (Заместитель) :

 Паттерн позволяет подставлять вместо реальных объектов специальные объекты-заменители. Эти объекты перехватывают вызовы к оригинальному объекту, позволяя сделать что-то до или после передачи вызова оригиналу.



Для чего используется:

 Контроль доступа к объекту, ленивая инициализация, кэширование, логирование обращений.



Пример использования:

 Виртуальные прокси для тяжёлых объектов, защищающие прокси для контроля доступа, удалённые прокси для сетевых объектов.



Простой пример из жизни:

 Как секретарь руководителя: вы не можете напрямую попасть к директору, секретарь контролирует доступ, проверяет полномочия и решает, пропускать ли вас.
