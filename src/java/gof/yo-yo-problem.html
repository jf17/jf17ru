Yo-Yo Problem (Проблема Йо-Йо) :

 Антипаттерн, возникающий при чрезмерном использовании наследования. Для понимания кода приходится постоянно перемещаться вверх и вниз по иерархии классов.



Признаки проблемы:

 Глубокая иерархия наследования, распределение логики по множественным уровням, сложность отслеживания выполнения кода.



Как избежать:

 Предпочтение композиции наследованию, ограничение глубины иерархии, использование интерфейсов, применение принципа единственной ответственности.



Простой пример из жизни:

 Как бюрократия в большой организации: чтобы понять простой вопрос, приходится ходить по множеству кабинетов вверх-вниз по иерархии, никто не может дать полный ответ.
