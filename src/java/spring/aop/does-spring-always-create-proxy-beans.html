Всегда ли Spring создаёт прокси бины ?

Нет, Spring создаёт прокси для бинов не всегда, а только в определённых случаях, когда это необходимо.
Прокси используются, когда Spring применяет некоторые аспекты к бинам, такие как транзакционность, безопасность или другие аспекты AOP (Aspect-Oriented Programming)


Рассмотрим основные случаи, когда Spring создаёт прокси:

  • Транзакционность Если у бина есть аннотация @Transactional, Spring создаёт прокси для управления транзакциями. Это позволяет оборачивать методы транзакционными аспектами, которые управляют началом, коммитом или откатом транзакции.

  • Безопасность (Spring Security)Если используется Spring Security и аннотации безопасности, такие как @Secured или @PreAuthorize, Spring создаёт прокси для добавления аспектов безопасности.

  • Событийное программирование (Event Listeners)Аннотация @EventListener требует создания прокси, чтобы бины могли реагировать на события.

  • Spring AOP (@Aspect) Если используется Spring AOP, чтобы добавить к методам кросс-концернные задачи (например, логирование, кэширование и т. д.), Spring создаёт прокси для бинов с аннотациями аспектов.

  • Lazy Initialization (отложенная инициализация) Если бин помечен как @Lazy, Spring может создать прокси для управления его инициализацией и загрузкой по требованию.


Когда прокси не создаются:

  • Если бин не использует аспекты, прокси создавать не нужно.
  • Прокси не создаются, если бин используется только для простого внедрения зависимостей без аспектов или если его метод вызывается напрямую.
