Жизненный цикл Bean в Spring :

  1. Инициализация контейнера. При запуске приложения Spring контейнер создаёт экземпляры всех бинов(BeanDefinition), определённых в конфигурации.
  2. Создание бина. Контейнер вызывает его конструктор или фабричный метод для создания экземпляра бина.
  3. Внедрение зависимостей. После создания бина контейнер внедряет зависимости, указанные в конфигурации.
  4. Настройка бина. После внедрения зависимостей контейнер вызывает методы инициализации бина.
  5. Использование бина. После настройки бина он готов к использованию в приложении.
  6. Уничтожение бина. Когда контекст приложения закрывается или бин больше не нужен, контейнер вызывает методы уничтожения бина. (только для singleton)
