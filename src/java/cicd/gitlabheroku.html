Java

1)

регистрируем домен второго уровня .
Например:
heroku-test-app-123-456

2)

добавляем файлик в корень проекта .gitlab-ci.yml
Оранжевым цветом помечено , то что нужно заменить в файле на свои значения!

Содержимое файлика: 

cache:
  paths:
    - .m2/repository

stages:
  - build
  - deploy

build:

  stage: build
  script:
    - 'mvn package'
  only:
    - dev

  artifacts:
    paths:
      - target/$JAR_FILE_NAME
  image: maven:3.3.9-jdk-8

deploy:
  stage: deploy
  image: ruby:latest
  script:
    - apt-get update -qy
    - apt-get install -y ruby-dev
    - gem install dpl
    - dpl --provider=heroku --app=$DOMAIN_NAME --api-key=$K8S_SECRET_API_KEY
  only:
    - dev


3)

Заходим в Gitlab .
Settings -> Ci/CD -> Environment variables
добавляем K8S_SECRET_API_KEY (который берем из панели управления Heroku)
добавляем DOMAIN_NAME (имя домена зарегистрированного в Heroku)
добавляем JAR_FILE_NAME (имя jar файла который получается после компиляции)

4)

коммитим в репу

5)

Заходим в Ci/CD
ждем когда Джоба закончит свою работу .

6)

проверяем результат
