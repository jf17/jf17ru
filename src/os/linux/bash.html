Bash

Bash — (от англ. Bourne again shell, каламбур «Born again» shell — «возрождённый» shell) — усовершенствованная и модернизированная вариация командной оболочки Bourne shell. 


script.sh

#!/usr/bin/env bash
echo 'Hello World!'

Сделать наш файл исполняемым:
chmod +x script.sh

Теперь можно запускать:
./script.sh

Thunar -> ПКМ на файле.sh -> Открыть с помощью -> Открыть в другом приложении... -> (внизу) Использовать команду -> sh

Ввод-вывод :
  • echo — выводит выражение или содержимое переменной (stdout), но имеет ограничения в использовании[3].
  • echo 'hello' > filename.txt — Перенаправление стандартного вывода в файл filename.txt.
  • echo 'hello' >> filename.txt — Перенаправление стандартного вывода в файл filename.txt , данные добавляются в конец файла. При отсутствии файла он создаётся
  • printf — команда форматированного вывода, расширенный вариант команды echo
  • read — «читает» значение переменной со стандартного ввода (stdin), в интерактивном режиме это клавиатура
  • read < filename.txt — Перенаправление стандартного ввода из файла filename.txt
  • 2>&1 — Перенаправление вывода ошибок на стандартный вывод .
  • command1 | command2 — Перенаправление вывода command1 на ввод command2 .

Файловая система :
  • touch filename.txt — Создать файл filename.txt .
  • mkdir directory-name — Создать папку directory-name .
  • ls — Просмотреть файлы в этом каталоге .
  • ls -lh — Просмотреть файлы ввиде списка и показать размеры файлов в Кб, Мб, Гб.
  • cd Directory1 — Перемещение в каталог Directory1 .
  • cd ~ — Перемещение в домашний каталог .
  • cd .. — Перемещение на один уровень выше .
  • pwd — выводит название текущего рабочего каталога (от англ. print working directory)
  • df -h — свободное место на дисках
  • df -h /home — свободное место в определенном каталоге

Чтение из файла :
  • cat filename.txt — Команда cat считывает файл, переданный как аргумент, и выводит его содержимое по стандартному каналу вывода .
  • head filename.txt — Команда head читает первые 10 строк любого переданного текста и выводит их по стандартному каналу.
  • tail filename.txt — Команда tail работает аналогично команде head, но читает строки с конца.
  • grep 'student` file1.txt — Команда grep занимается поиском переданной строки в указанном файле. -i Отключение чувствительности к регистру

Коннекторы :
  • command1 && command2 — Первая команда исполняется всегда, вторая — только в случае успешного завершения первой .
  • command1 || command2 — Первая команда исполняется всегда, вторая — только в случае неудачного завершения первой .
  • command1 ; command2 — Команды исполняются всегда .

Процессы :
  • kill — посылает сигнал завершения процессу.
  • jobs — показывает процессы висящие в foreground или background.
  • fg — Ctrl+Z - перевести процесс в фоновый режим. foreground(на переднем плане) процесс висит в фоновом режиме(не работает)
  • bg — перевести процесс в background (программа будет работать ).
  • ps — Команда выводит информацию о запущенных процессах. (ID процесса (PID) , тип терминала (TTY),время работы процесса (TIME),имя команды, запустившей процесс (CMD) )
  • top — консольная команда, которая выводит список работающих в системе процессов и информацию о них. По умолчанию она в реальном времени сортирует их по нагрузке на процессор .

Работа с пакетами :
  • sudo apt-get update — Команда update используется для повторной синхронизации индекса файлов пакета из своих источников, указанных в файле /etc/apt/sources.list .
  • sudo apt-get upgrade — Команда upgrade используется для обновления всех установленных пакетов программного обеспечения в системе .
  • sudo apt-get install имяпакета — Команда install отслеживает один или несколько пакетов для установки или модернизации.
  • sudo apt-get download имяпакета — Скачать любой пакет без установки.
  • sudo apt-get remove имяпакета — Чтобы деинсталлировать пакеты программного обеспечения, не удаляя их конфигурационных файлов (для последующего повторного использования той же конфигурации), используйте команду remove.
  • sudo apt-get purge имяпакета — Для удаления программных пакетов, включая их конфигурационные файлы, используйте purge.
  • sudo apt-get --download-only source имяпакета — Скачать только исходный код пакета.
  • sudo apt-get source имяпакета — Скачать и распаковать пакет.
  • sudo apt-get --compile source имяпакета — Скачать, распаковать и скомпилировать исходный код одновременно .
  • sudo apt-get build-dep имяпакета — Команда ищет локальные репозитории в системе и устанавливает зависимости для сборки пакета. .
  • sudo apt-get check — Команда используется для обновления кэша пакетов и проверяет его на предмет сломанных зависимостей.
  • apt-cache showpkg имяпакета — Используйте showpkg для проверки зависимостей для определенных пакетов программного обеспечения .
  • apt-cache show имяпакета — Если вы хотите проверить информацию о пакете и получить краткое описание (номер версии, проверка суммы, размера, установленного размера, категории и т.д.), используйте ' показать команду show.
  • apt-cache search имяпакета — поиск пакета по части названия или описания. Поддерживает регулярные выражения.
